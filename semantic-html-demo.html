Document Structure Elements

\n \n
\n

<header> - Page/Section Header

\n

Contains introductory content, navigation aids, or branding elements.

\n
\n<header role=\"banner\">\n <h1>Site Title</h1>\n <nav>...</nav>\n</header>\n
\n
\n\n
\n

<nav> - Navigation Links

\n

Contains navigation links to other pages or sections within the current page.

\n
\n<nav role=\"navigation\" aria-label=\"Primary navigation\">\n <ul>\n <li><a href=\"#home\">Home</a></li>\n <li><a href=\"#about\">About</a></li>\n </ul>\n</nav>\n
\n
\n\n
\n

<main> - Main Content

\n

Contains the primary content of the document. Only one per page.

\n
\n<main role=\"main\">\n <!-- Primary page content -->\n</main>\n
\n
\n\n
\n

<footer> - Page/Section Footer

\n

Contains footer information like copyright, contact info, or related links.

\n
\n<footer role=\"contentinfo\">\n <p>© 2024 CodeQuest</p>\n</footer>\n
\n
\n
\n\n \n
\n

Sectioning Elements

\n \n
\n

<section> - Thematic Grouping

\n

Groups related content together with a common theme or purpose.

\n
\n<section aria-labelledby=\"progress-heading\">\n <h3 id=\"progress-heading\">Learning Progress</h3>\n <!-- Progress content -->\n</section>\n
\n
\n\n
\n

<article> - Standalone Content

\n

Self-contained content that could be distributed independently.

\n
\n<article>\n <header>\n <h3>Blog Post Title</h3>\n <time datetime=\"2024-01-15\">January 15, 2024</time>\n </header>\n <p>Article content...</p>\n</article>\n
\n
\n\n
\n

<aside> - Complementary Content

\n

Content that is tangentially related to the main content.

\n
\n<aside role=\"complementary\" aria-label=\"Related links\">\n <h4>Related Resources</h4>\n <ul>...</ul>\n</aside>\n
\n

Inline Semantic Elements

\n

Use strong for important text, em for emphasis, mark for highlighting, <code> for code snippets, and Ctrl+C for keyboard input.

\n
\n<p>Use <strong>strong</strong> for importance, \n<em>em</em> for emphasis, \n<mark>mark</mark> for highlighting.</p>\n

Semantic HTML Best Practices

\n \n
\n

✅ Do's

\n
    \n
  • Use heading hierarchy (h1 → h2 → h3) properly
    • \n
  • Include alt attributes for all images
    • \n
  • Use semantic elements instead of generic divs when possible
    • \n
  • Associate labels with form inputs using for/id
    • \n
  • Use ARIA attributes to enhance semantics when needed
    • \n
  • Include lang attribute on html element
    • \n
  • Use proper list markup for grouped items
    • \n
\n
\n\n
\n

❌ Don'ts

\n
    \n
  • Don't skip heading levels (h1 → h3)
    • \n
  • Don't use multiple h1 elements on the same page
    • \n
  • Don't use divs for everything when semantic elements exist
    • \n
  • Don't forget alt text for images
    • \n
  • Don't use placeholder text as labels
    • \n
  • Don't use tables for layout purposes
    • \n
  • Don't rely solely on color to convey information
    • \n
